# Тестовый проект интернет магазина
## Реализованный функционал
- API сервиса для управления заказами
- Фабрики и сидеры для сущностей Customer, Product
- Кэширование данных с помощью Redis
- Валидация остатков в процессе создания заказа
- Оптимизация производительности, кэш, индексы в БД
- Ограничение по ip на количество запросов для создания заказа
- Вызов Job для экспорта данных по заказу во внешнюю систему во время смены статуса на "подтвержден"
- Фича тесты основного функционала
- Таблица 'order_exports' для отслеживания статуса экспорта заказов с помощью Job

## Установка
### Склонируем репозиторий
```
git clone https://github.com/YakovLaptev/test-orders.git
cd test-orders
```
### Соберем контейнеры и запустим докер
```
docker-compose up -d
```
### Подготовим Laravel приложение к работе
#### Зайдем внутрь Laravel контейнера
``` 
docker exec -it orders-app sh
```
#### Сконфигурируем приложение и запустим обработку очередей 

``` 
composer setup
``` 

_*На сообщение `One new database migration has been published. Would you like to run all pending database migrations? (yes/no) `, ответ `yes`_
``` 
php artisan queue:work
```
_*Отредактировать `.env`, указав свои значения если необходимо_

_*Заполнить `ORDER_EXPORT_URL` в `.env`, указав URL внешнего сервиса для отправки данных по заказу_

#### Запустим тесты
``` 
docker exec -it orders-app sh

composer test
```
### Можно отправлять запросы на API и запускать тесты
**API Spec:**

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/products` | Список товаров. Фильтрация по category, поиск по name/sku, пагинация. |
| POST | `/orders` | Создание заказа (customer_id, массив items). Валидация остатков, атомарное списание stock через транзакцию. |
| GET | `/orders` | Список заказов. Фильтры: status, customer_id, диапазон дат. Items и customer подгружаются без N+1. |
| GET | `/orders/{id}` | Детали заказа. |
| PATCH | `/orders/{id}/status` | Смена статуса с валидацией допустимых переходов. |
